Technology Stack Research Report
1. Technology Stack Overview
Project Requirements
The technology stack must support:
Schwab API integration
Automated daily financial data collection
Historical market data storage
Backend API development
Web/mobile user interface
Future expansion into quantitative analysis and machine learning
The MVP prioritizes:
Fast development
Learning backend engineering practices
Reliable functionality
Simple deployment

2. Programming Language Research
Selected Technology: Python
Description
Python is a high-level programming language widely used in:
Backend development
Data engineering
Financial analysis
Machine learning

Advantages
Advantage
Explanation
Strong data ecosystem
Libraries such as pandas, NumPy, and SciPy support financial analysis
API integration
Simple HTTP communication with external APIs
ML compatibility
Easy future integration with machine learning models
Developer productivity
Faster MVP development


Disadvantages
Disadvantage
Explanation
Lower execution speed
Slower than C++ for performance-critical systems
Scaling requires architecture improvements
Large systems may require additional optimization


Decision
Selected: Python
Reason:
The project focuses on financial data collection, storage, and analysis rather than ultra-low-latency trading systems.

Description
FastAPI is a modern Python framework used to build REST APIs.
The backend API will serve as the communication layer between:

    |

    |

Advantages
Advantage
Explanation
Python integration
Works naturally with data processing libraries
Fast development
Minimal setup required
Automatic documentation
Generates API documentation automatically
Type validation
Helps prevent invalid requests


Disadvantages
Disadvantage
Explanation
Requires selecting additional components
Authentication and database tools must be added separately


Decision
Selected: FastAPI
Reason:
FastAPI provides a lightweight but professional backend framework suitable for learning backend architecture and building an MVP.

5. Database Research
Initial Database: SQLite
Future Database: PostgreSQL / Supabase

SQLite
Description
SQLite is a lightweight relational database stored locally as a single file.

Advantages
Advantage
Explanation
Zero setup
No database server required
Easy development
Works immediately on local machine
Simple deployment
Database can be included with application
Sufficient capacity
Handles MVP dataset requirements


Disadvantages
Disadvantage
Explanation
Limited concurrent users
Not designed for many simultaneous writes
Scaling limitations
Requires migration for larger systems


PostgreSQL / Supabase Future Migration
Description
PostgreSQL is a production-grade relational database.
Supabase provides:
Hosted PostgreSQL
Authentication
Database management tools
APIs

Reason for Future Migration
Move when the application requires:
Multiple users
Cloud deployment
Larger datasets
Higher reliability

Database Decision
MVP:
FastAPI
   |
SQLAlchemy
   |
SQLite

Future:
FastAPI
   |

   |
PostgreSQL / Supabase


6. Database ORM Research
Selected Technology: SQLAlchemy
Description
SQLAlchemy allows Python applications to interact with databases using Python objects instead of writing every SQL query manually.

Advantages
Cleaner backend structure
Easier database migration
Industry-standard Python ORM
Separates database logic from application logic

Decision
Selected:
SQLAlchemy
Reason:
Allows SQLite usage now while keeping migration to PostgreSQL simple.
